# Feedback

Feedback captures what the engineer learned after validating or fixing an issue.

Navigation path: AI Chat -> Developer feedback.

Steps:
1. Run an investigation.
2. Select Helpful, Not Helpful, Partially Correct, Wrong Root Cause, Missing Evidence, or Needs DBA Review.
3. Enter actual root cause.
4. Enter actual fix applied.
5. Enter SQL or procedure changed.
6. Enter test cases and proof of fix.
7. Enter rollback information and notes.
8. Submit for DBA or lead approval.

Warnings:
- Feedback remains pending until approved.
- Do not submit secrets or passwords in feedback fields.
